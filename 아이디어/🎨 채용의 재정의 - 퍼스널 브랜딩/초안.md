# 채용 패러다임의 변화
1. AI로 인한 생산성 폭발
	- 한 사람이 다룰 수 있는 범위가 넓어짐 -> 대규모 인력 투입 필요X
	- 기업은 많은 사람보다 적합한 사람 한명/한 팀을 원함
2. 채용에서 영입으로
	- 기존 : 기업이 공고 -> 지원자 몰림 -> 서류전형 -> 합격
	- 미래 : 기업이 탐색 -> 짝맞는 인재,팀을 픽업 -> 빠른 계약 합류
3. 퍼스널 브랜딩 필수화
	- 내가 누구인가를 작품 성과 가치관으로 전시해야됨
	- 단순 스팩이 아닌 임팩트와 철학이 경쟁력

# 첫번째 인재 타겟 -> 개발자
- 디자이너, 일러스트레이터의 전시 공간은 많지만 개발자는 적음 - 첫번째 타켓 개발자
- 개발자들 특성상 깃허브, 블로드, 포트폴리오 등 자기 브랜딩 습관이 있음

# 서비스 콘셉트 - 인재 전시관 & 영입 허브
- 개인,팀이 자신을 브랜드로 전시
- 기업은 탐색,필터,스타우트 방식으로 영입
- 잡코리아, 사람인이 아니라 “Dribbble+LinkedIn+Hired” 혼합 모델

### 1. 조각난 브랜딩을 하나로 통합
- 깃허브 ->코드 / 링트드인 -> 경력 / 블로그 -> 글.생각 전부 흩어져있음
- 내 브랜딩의 원본 프로필이 되는 것이 목표 -> 나를 알리는 시발점
	- 깃허브 리포 자동 임베드
	- 블로그 포스트 자동 싱크
	- 컨퍼런스 발표/유튜브 동영상 연동

### 2. 성과,임팩트 지표 시각화
- 정량화된 성과 대시보드
	- 응답속도 2s -> 300ms
	- 배포주기 2주 -> 1일 단축

### 3. 가치,철학 컬처핏 기반 필터
- 기술, 경력만 강조하는 것이 아니라 나는 어떤 가치관/환경/프로젝트에 열려있는가를 전시

### 4. 팀 단위 전시 & 영입
- 팀단위 프로필 지원 : 기업은 사람1명이 아니라 바로 투입 가능한 팀까지 픽업 가능

### 5. 실시간 가용성 & 오퍼
- 즉시 시작 가능/ 주 20시간 / 3개월 단위 계약

### 6. 살아있는 문서, 자동 갱신 기능
1. **GitHub 연동**
	- 커밋/PR/스타 활동을 주기적으로 크롤링 or API 연동
	- “지난주 활동”을 그래프(잔디)로 보여줌
	- 주요 리포 업데이트 자동 노출 (README 일부 + 링크)
 1. **블로그 연동**
	- RSS/Atom 피드 기반으로 글 제목 + 썸네일 + 미리보기 자동 수집
	- Notion/Velog/Tistory/Medium/개인 블로그 모두 RSS 지원 가능
	- 최신 글이 자동 카드로 “활동” 영역에 추가
 2. **패키지/프로젝트 활동**
	- npm, PyPI, DockerHub 등 배포된 패키지 다운로드 수 자동 업데이트
	- 오픈소스 이슈/컨트리뷰션 기록 자동 노출
3. **발표·커뮤니티 활동**
	- YouTube 채널 연동 → 발표 영상 자동 임베드
	- SlideShare/Notion 발표자료 자동 embed
4. **자동 요약 카드**
	- AI Summarizer → 최근 활동을 주기적으로 요약해서 “이번 달 ○○ 활동” 같은 하이라이트 제공

## 👤 개발자 프로필 구조
## 1. **Hero 영역**
### 필드
- `name` / `handle`
- `specialization`: Frontend / Backend / AI / Infra / Fullstack
- `tagline`: 한 줄 임팩트 문구
- `location`, `availability`(즉시/프로젝트 단위/계약 가능 여부)

### UI
- 상단 프로필 카드:
    - [프로필 사진/아바타]
    - 이름(핸들네임)
    - 전문 분야 배지 (예: `Frontend Engineer`)
    - 한 줄 태그라인
    - 상태 표시(예: “2주 내 합류 가능”)
## 2. **코드/프로젝트 전시**
### GitHub 연동
- Stars, Forks 수 → 상위 Repo 자동 노출
- Commit Activity 그래프(최근 12개월)
### 프로젝트 카드 구조
- `title`: 프로젝트명
- `problem`: 문제 정의 (한두 줄)
- `role`: 내 역할 (예: API 설계 60%, DevOps 40%)
- `stack`: 사용 스택(React, Spring, AWS 등)
- `outcome`: 성과 (예: 응답속도 2s → 300ms)
- `links`: Repo / Demo / Blog
    
## 3. **스킬 매트릭스**
### 필드
- `languages`: JS(5년), TS(3년), Python(2년)
- `frameworks`: React(고급), Spring Boot(중급)
- `tools`: Docker, Jenkins, Cypress
- `extraMetrics`: 테스트 커버리지 개선 경험, 배포 자동화 경험
### UI
- 바 차트 / 원형 그래프
- “자주 쓰는 언어 Top 3” 하이라이트
- 부가 지표를 아이콘/텍스트로 표시
## 4. **가치·선호**
### 필드
- `ossContrib`: 오픈소스 기여 여부(Repo 링크)
- `preferredDomains`: 핀테크, SaaS, 게임, ESG 등
- `workStyle`: 원격 / 하이브리드 / 온사이트
- `companySize`: 스타트업/중소기업/대기업 선호
    
### UI
- 태그형 뱃지 + 강조 아이콘
- 기업이 바로 필터링 가능
## 5. **신뢰 시그널**
### 필드
- `endorsements`: 추천사 (멘토/팀리드/동료)
- `blogLinks`: 기술 블로그/Medium/Velog 링크
- `talks`: 컨퍼런스/밋업 발표 영상 링크
### UI

- 추천사 = 카드형 (작성자 이름, 직책, 코멘트 한 줄)
- 블로그 = 썸네일/최근 글 제목 3개
- 발표 = YouTube/슬라이드 임베드

좋은 문제의식이야. 사실상 “채용 플랫폼”의 패러다임을 바꾸려는 아이디어인데, 이미 LinkedIn이나 국내 원티드/로켓펀치 같은 곳이 존재하지? 하지만 네가 말한 건 훨씬 더 **‘포트폴리오 퍼스널 브랜딩 플랫폼’**에 가까워.  
즉, 이력서를 올려두고 기다리는 게 아니라, **내가 누구인지 ‘전시’하고 기업이 와서 픽업하는 구조**.

여기에 독특한 아이덴티티를 줄 수 있는 방향 몇 가지 제안해볼게.

## 🎨 차별화 아이덴티티 아이디어

### 1. **“채용”이 아니라 “전시/마켓플레이스”**

- 콘셉트를 **Artstation / Behance + LinkedIn** 느낌으로 가져가.
    
- 기업은 **“인재 마켓”**에 들어와서 탐색하는 소비자,
    
- 근로자는 **“작품/브랜드”**로서 자신을 올려두는 창작자.  
    👉 “채용 공고 사이트”가 아니라, **“사람 마켓 / 갤러리”** 정체성.
    

---

### 2. **환경처럼, 사회적 가치 반영**

- 단순 포트폴리오가 아니라 **내가 지향하는 가치**를 함께 기록.  
    예:
    
- “나는 지속가능성 프로젝트만 참여합니다.”
    
- “나는 사회적 기업/임팩트 스타트업에만 열려있습니다.”  
    👉 기업이 단순 스펙이 아니라 **가치/철학** 기반으로 사람을 찾게 됨.
    

---

### 3. **멀티미디어 자기표현**

- 텍스트+PDF 이력서 대신,
    
    - **짧은 영상 소개**
        
    - **라이브 포트폴리오(코드 데모, 디자인 인터랙션, 실시간 링크)**
        
    - **협업 스타일을 드러내는 테스트/성향 프로필**  
        👉 기업이 “이력서 읽기” 대신 “사람을 체험”하게 되는 플랫폼.
        

---

### 4. **기업 → 사람으로 흐름 반전**

- 기존: 기업이 공고 → 근로자가 지원
    
- 제안: 근로자가 스스로를 공개 → 기업이 “Connect / Offer” 버튼 클릭  
    👉 기업이 사냥하듯 “좋은 사람 찾기”에 맞춰져 있음.  
    이를 **“역채용(Reverse Hiring)”** 모델로 정체화.
    

---

### 5. **게임화·커뮤니티성**

- “이번 달 가장 많이 제안받은 프로필 Top 10”
    
- “디자인/마케팅/개발 각 분야 인기 전시작”
    
- “추천 태그 기반 큐레이션”  
    👉 단순 DB가 아니라 **브랜딩 + 커뮤니티**.
    

---

### 6. **국내 문화에 맞춘 차별화**

- LinkedIn은 글로벌하지만 한국에선 여전히 구인/구직은 사람인·잡코리아·원티드 쪽.
    
- 네 서비스는 **“포트폴리오 퍼블릭 갤러리”**를 전면에 내세우고, **기업 탐색 → 바로 Offer**까지 이어지도록 UX 차별.

## 🔑 요약

- 이름/아이덴티티를 “구직 사이트”로 두지 말고, **“인재 전시관 / 사람 마켓”** 같은 방향으로 잡아.
    
- 채용 = 스펙 매칭이 아니라, **“브랜딩된 개인을 찾아가는 과정”**으로 재정의.
    
- **나의 가치 + 나의 작업 + 나의 모습**을 종합적으로 보여주는 공간.
